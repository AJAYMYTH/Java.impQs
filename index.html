1. Explain Java Server Pages (JSP) with Example

Definition: JSP is a server-side Java technology that allows developers to embed Java code within HTML to create dynamic web pages.
Key Concepts:
  • Translated into Servlets by the JSP container on first request
  • Components: Directives (<%@ %>), Scriptlets (<% %>), Expressions (<%= %>), Declarations (<%! %>), Standard Actions
  • Lifecycle: Translation → Compilation → Initialization → Execution → Cleanup
Example: 
<%@ page language="java" contentType="text/html; charset=UTF-8" %>


  
Welcome, <%= request.getParameter("user") %>!

  
Server Time: <%= new java.util.Date() %>
Significance: Separates presentation from business logic, simplifies dynamic HTML generation, supports JSTL/EL, and acts as the View layer in MVC architecture.

3. Explain Insert Operation in JDBC with an Example

Steps: Load driver → Establish connection → Prepare SQL → Set parameters → Execute → Close resources
Key Concepts: Use PreparedStatement to prevent SQL injection and improve performance via precompilation.
Example: 
String sql = "INSERT INTO employees (name, dept) VALUES (?, ?)";
try (Connection conn = DriverManager.getConnection(url, user, pass);
     PreparedStatement pstmt = conn.prepareStatement(sql)) {
    pstmt.setString(1, "Ajay");
    pstmt.setString(2, "IT");
    int rows = pstmt.executeUpdate();
    System.out.println(rows + " row(s) inserted.");
}
Significance: executeUpdate() returns affected row count. Try-with-resources ensures automatic cleanup. Parameter binding enhances security and performance.

4. Explain Delete Operation in JDBC with Example

Mechanism: Uses DELETE FROM table WHERE condition executed via PreparedStatement.
Key Points: Always include a WHERE clause to avoid accidental full-table deletion. Returns integer count of deleted rows.
Example: 
String sql = "DELETE FROM employees WHERE id = ?";try (Connection conn = DriverManager.getConnection(url, user, pass);
     PreparedStatement pstmt = conn.prepareStatement(sql)) {
    pstmt.setInt(1, 105);
    int deleted = pstmt.executeUpdate();
    if (deleted > 0) System.out.println("Record deleted successfully.");
}
Significance: Supports transactions (conn.setAutoCommit(false)). Proper error handling and parameterization prevent data loss and injection attacks.

5. Explain List in Java Collection Framework with Types & Example

Definition: List is an ordered collection that allows duplicate elements and provides positional access via index.
Types:
  • ArrayList: Resizable array, fast random access, slower mid-list insertions
  • LinkedList: Doubly-linked list, fast insertions/deletions, slower random access
  • Vector/Stack: Legacy thread-safe classes (prefer ArrayList + Collections.synchronizedList())
Example: 
List list = new ArrayList<>();
list.add("Java"); list.add("Python"); list.add("Java");
System.out.println(list.get(1)); // Output: Python
Significance: Implements Iterable, Collection, List. Supports Iterator, ListIterator, subList(), and Collections.sort().

Components:

Hello, " + name + "
");

Significance: Enables dynamic content generation. Headers manage caching, security, sessions. Java's API abstracts low-level HTTP parsing.

9. Explain Select Operation in JDBC with Example

Mechanism: Retrieves data using SELECT via executeQuery(), returning a ResultSet.
Steps: Connect → Prepare → Execute → Iterate ResultSet → Extract data → Close
Example: 
String sql = "SELECT name, salary FROM employees WHERE dept = ?";
try (Connection conn = DriverManager.getConnection(url, user, pass);
     PreparedStatement pstmt = conn.prepareStatement(sql)) {
    pstmt.setString(1, "IT");
    ResultSet rs = pstmt.executeQuery();
    while (rs.next()) {
        System.out.println(rs.getString("name") + ": " + rs.getDouble("salary"));
    }
}
Significance: ResultSet cursor starts before first row. Use rs.next() to traverse. Always close ResultSet to prevent memory leaks.
